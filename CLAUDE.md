# OvertPoly

## Environment setup 
Before doing anything, activate the OvertPoly Julia

## Rules
- Never run tests directly in bash, add them to the corresponding test script and run that script instead.
- Whenever you edit a file, write a quick summary explaining changes made 
- Whenever you run tests, generate a concise report that describes the tests, inputs, expected outputs, actual outputs, and conclusions
  - Write this report to a markdown file in the tests/reports folder. The title of the report should be follow the format of "report_{insert task name}_{date in yyyy-mm-dd-hh-mm format}.md"
- Commit (and push) your changes every 3 hours and use a descriptive commit message that starts with "Claude:{insert message here}"
